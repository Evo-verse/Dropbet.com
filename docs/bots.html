Casino Bot Service Description

1. General Information

  • The bot service allows automating gaming activity in the casino using any accounts created on the platform.
  • Bots can imitate the behavior of real players, following predefined strategies and schedules.
  • The system supports strategy extensions, enabling the creation of new behavior patterns without changing the core logic.
  • Game activity generation is only available for enabled slot games.
  • All bot bets go through the system’s internal aggregator, bypassing the external slot aggregator, which eliminates any commission fees.

2. Bot Creation and Configuration

2.1. Account Assignment

Any existing account on the platform can be designated as a bot.

2.2. Schedule

For each bot, the following parameters are configured:

  • Days of the week when the bot will be active.
  • Initial start time in UTC format.
  • Randomized session duration, calculated based on the bot’s personality configuration.
  • Start time offset — each new start time is generated with a ±1 hour dispersion from the personal start time.
  • Breaks during play — the bot may take a random break of up to 1 hour during its session.

3. Bot Types

The bot type defines its betting level and session duration:

Whale ~6 hours High bets

4. Behavioral Patterns

4.1. Casual

  • Rarely changes bets.
  • Slight bet increase after wins.
  • Bet decrease after a losing streak.
  • Switches slot after a long series of losses.

4.2. Regular/Normal

  • Keeps bet size at the level of a single line.
  • Plays on the same slot for the entire session.
  • Switches to another slot only after a big win or a large number of bets on the same slot.

4.3. Chaser

  • Increases bet after a loss to recover losses.
  • May change slot after prolonged losses at high stakes.
  • Switches slot after significant wins.

4.4. Highroller

  • Keeps bets within a fixed range.
  • Plays until hitting a high multiplier.

4.5. Trickster

  • Each new bet adopts the behavior of one of the available strategies (random selection).

5. Bot Decision-Making Logic

During a session, the bot:

  • Evaluates the current game state.
  • Decides whether to:
    • Increase the bet.
    • Decrease the bet.
    • Change to another slot from the available list.
  • Follows the behavioral pattern that defines its core decision-making strategy.
