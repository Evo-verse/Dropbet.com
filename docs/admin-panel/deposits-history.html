10. Deposits History

The Deposits History section allows admins to view and track all incoming deposit transactions made by users across supported networks.

Displayed Information:

  • User – Username and avatar of the depositor.
  • Network – Blockchain network used (e.g., ETH, SOL, BSC, LTC).
  • Transaction Hash – Clickable link to view the transaction on the blockchain explorer.
  • From Address – Sender’s wallet address.
  • Amount – Amount deposited, shown in cryptocurrency and approximate USD value.
  • Status –
    • Complete – Deposit confirmed and credited to the user’s account.
    • Failed – Deposit was detected but could not be credited (e.g., network issues, unsupported token type, incorrect amount).
  • Referral Code – Referral code associated with the deposit (if any).
  • External ID – Internal tracking ID for the deposit request.

Admin Actions:

No manual approval is required for deposits — they are processed automatically after blockchain confirmation.

Failed deposits require manual investigation; admins can coordinate with tech support to retry or manually credit funds.
