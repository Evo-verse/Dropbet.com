2. Game Management

The Game Management module allows administrators to monitor, configure, and analyze all games available on the platform, including both third-party aggregated slots and in-house games. It provides performance metrics, provider-level data, and individual game configuration tools.

2.1 Game Stats

The Game Stats page presents global performance indicators for all games within the selected date range.

Key metrics displayed:

  • GGR Games – Gross Gaming Revenue from all games in the period.
  • Total Bet Amount – Combined value of all wagers placed.
  • Total Bet Count – Number of bets made.
  • Average Bet – Mean bet size calculated from total wagers and bet count.

Filters and Controls:

  • Date range selector (Last 24h, custom start/end date).
  • Category filter to choose which stats to display.

2.2 Provider Stats

The Provider Stats page shows performance broken down by each game provider.

Columns:

  • Provider Slug – Internal identifier for the provider.
  • Turnover – Total wager amount for games from this provider.
  • Bet Count – Number of bets placed.
  • Average Bet – Mean wager size per bet.
  • GGR – Gross Gaming Revenue for that provider (positive in green, negative in red).

Filters and Controls:

  • Date range selector.
  • Filter by date or other criteria.

2.3 In-House Game Pages

Each in-house game has its own dedicated admin page (e.g., Plinko, Roulette, Mines, Keno, Limbo, Speed Roulette, Blackjack, Dice, Monkey Run). These pages share a consistent structure:

Metrics

  • GGR Games – Gross Gaming Revenue generated by this game.
  • Total Bet Amount – Sum of all wagers.
  • Total Bet Count – Number of bets.
  • Average Bet – Mean wager size.

Filters

  • Time period selection (Last 24h or custom range).
  • Status filter (e.g., active, settled, refunded).

Data Table

Displays individual bets placed in the selected timeframe, including:

  • User ID / Username.
  • Game name.
  • Bet amount.
  • Multiplier achieved.
  • Payout amount.
  • Date of bet.
  • Additional info link for detailed view.

2.4 Game Configuration

For configurable in-house games (e.g., Plinko), the Change Game Configuration button opens a settings modal where admins can set:

  • Min Bet – Minimum allowed bet size.
  • Max Bet – Maximum allowed bet size.
  • Max Profit – Maximum payout allowed per round.
  • Manual MS – Delay in milliseconds for manual play.
  • Auto MS – Delay in milliseconds for autoplay.

Changes are applied after clicking Submit and saved to the game configuration backend.

2.5 In-House Games Chart

The In-House Games Chart provides a visual breakdown of selected games’ performance.

Admins can:

  • Select multiple games to compare.
  • Choose data type to display:
    • Total Wagered
    • Total Payouts
    • Total Bets
  • View percentage contribution of each game in a color-coded pie chart.
  • See absolute numbers for each game alongside percentages.
