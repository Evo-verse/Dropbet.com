7. Bets History

The Bets History section is used to review all bets placed on the platform, making it essential for monitoring gameplay activity, verifying results, and investigating user claims.

Filters & Search

Admins can refine the search using:

  • Date Range – select from “All time” or custom start/end dates.
  • Bet Amount – set minimum and maximum bet values.
  • Payout – filter results by payout amount range.
  • Multiplier – find bets within specific multiplier ranges.
  • User ID / Game ID – directly locate bets from a specific player or game.
  • Status – filter by bet status (e.g., completed, pending).

These filters help narrow down large datasets for specific cases, such as big wins, suspicious activity, or jackpot hits.

Bet Records Table

The results display:

  • User – player username and avatar.
  • Game – game type played (clickable link to provider/game info).
  • Bet – amount wagered.
  • Multiplier – win multiplier achieved.
  • Payout – net win amount (positive or negative).
  • Date – exact timestamp of the bet.
  • More Info – opens detailed bet round data, including server response for verification.
