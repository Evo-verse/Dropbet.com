KYC Limits Management

Overview

The KYC Limits Management page allows administrators to define maximum withdrawal limits for each KYC (Know Your Customer) verification level. This ensures withdrawal permissions are aligned with the user’s verified identity status.

Page Layout & Features

1. KYC Level

  • Displays all available KYC tiers (e.g., LEVEL_0 to LEVEL_4).
  • Each level corresponds to a different verification stage:
    • LEVEL_0 – No or minimal verification.
    • LEVEL_1+ – Increasing levels of verification such as ID, address proof, and source of funds.

2. Unlimited Withdrawal

  • Toggle switch to allow users in that KYC tier to withdraw without limits.
  • If enabled, the Max Withdrawal Amount field becomes inactive for that tier.

3. Max Withdrawal Amount (USD)

  • Defines the maximum withdrawal amount allowed for that KYC level.
  • If Unlimited Withdrawal is disabled, this field sets the exact withdrawal cap.
  • Values are entered in USD.

4. Save Changes

  • After making any adjustments, click Save changes to apply updates immediately for all users in the affected tier.

Challenges

Overview

The Challenges page allows admins to create, manage, and review gaming challenges for players. It is divided into two main sections: Open (active challenges) and Finished (completed challenges).

1. Open Challenges Tab

Displays all currently active challenges with the following columns:

  • Admin – Username of the admin who created the challenge.
  • Game – The linked game for the challenge.
  • Slug – Unique identifier used for the challenge link.
  • Challenge – Description of the objective (e.g., “First to hit 250x on Gates of Olympus”).
  • Hidden – Indicates if the challenge is visible to players.
  • Prize – Reward for completing the challenge.
  • Candidates – Number of players participating.
  • Created At – Date and time created.
  • Actions – Manage challenge (view/edit/delete).

Buttons & Filters

  • Add Challenge – Opens form to create a new challenge.
  • Search / Filters – Search by slug, filter by game or admin.

2. Finished Challenges Tab

Displays completed challenges with an additional column:

  • Winner – Shows the player who won the challenge.

Admins can use Search Winner to find past winners or review performance.

3. Bets Challenge View

When viewing a challenge’s bets, the table includes:

  • User – Player’s username.
  • Game – Game where the bet occurred.
  • Bet – Amount placed.
  • Multiplier – Multiplier achieved.
  • Payout – Total winnings.
  • Settled At – Date/time settled.
  • Actions – View Details or Award prize manually.

Challenge Creation Fields

  • Min bet amount (DBC) – Minimum bet required for entry.
  • Reward amount – Prize value in DBC or equivalent currency.
  • Game slug – Unique identifier for the game (must match system slug).
  • Min bet multiplier – Minimum multiplier needed to qualify.
  • Hidden challenge – Toggle Yes/No for visibility.
  • Challenge image – Upload image to display with the challenge.

Buttons

  • Upload image – Attach image file.
  • Cancel – Discard form without saving.
  • Confirm – Save and activate challenge.
